1. Опис кроків, які були виконані

Спочатку я встановила Node.js, після чого перезапустила термінал, щоб система
розпізнала команди node та npm. У терміналі створила новий Vue-проєкт командою npm create vue@latest vue-lab
і у меню вибрала потрібні опції: увімкнула TypeScript, Vue Router, ESLint і Prettier.
Після створення проєкту перейшла до його папки (cd vue-lab),
встановила всі залежності (npm install) та запустила сервер розробки командою npm run dev,
відкривши проєкт у браузері.У практичній частині я створила нову сторінку Contact, додала компонент ContactView.vue,
зареєструвала для неї маршрут у router/index.ts, додала посилання у меню, а потім відкрила її в браузері.
Я створила компоненент UserCard.vue з типізованими пропсами і додала його на сторінку Contact,
щоб продемонструвати використання TypeScript у Vue.
Після перевірки сторінка успішно відображає інформацію без перезавантаження — завдяки Vue Router.

3. Висновки: що дає використання TypeScript та Vue Router у проєкті?

Під час виконання роботи я зрозумів, що TypeScript значно спрощує розробку,
тому що він одразу показує помилки типів і допомагає уникати багатьох проблем ще до запуску програми.
Це робить код зрозумілішим, передбачуваним і зручним у підтримці. Vue Router, у свою чергу,
дозволяє легко створювати багатосторінкову структуру в межах одного застосунку.
Завдяки йому можна перемикатися між сторінками без перезавантаження браузера,
організовувати маршрути, додавати нові сторінки.
