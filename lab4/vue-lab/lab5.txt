1. Опишіть кроки, які ви виконали для інсталяції та налаштування Tailwind CSS.
2. Поясніть, як ви реалізували умовні стилі (для isActive) у UserCard.vue за допомогою класів Tailwind.
3. Які переваги та недоліки utility-first підходу (Tailwind) ви помітили порівняно з <style scoped>?


1. Спочатку встановила Tailwind CSS та плагін для Vite командою: npm install -D tailwindcss @tailwindcss/vite
Потім налаштувала vite.config.ts, додавши плагін tailwindcss() до масиву plugins.
Створила файл src/assets/css/index.css з рядком @import "tailwindcss";.
Підключила стилі в main.ts, додавши імпорт ./assets/css/index.css.
Після цього перезапустила сервер командою npm run dev.

2. Використала директиву :class з тернарним оператором. Якщо isActive дорівнює true,
застосовуються класи border-green-500 bg-green-50, якщо false — border-gray-300 bg-white.
Так само для тексту статусу: коли активний — text-green-700, коли ні — text-gray-500.
Це дозволило змінювати вигляд картки залежно від стану користувача без написання окремих CSS-класів.

3. Під час роботи з Tailwind CSS я помітила кілька переваг порівняно з <style scoped>.
Розробка стала набагато швидшою, оскільки не потрібно придумувати назви для CSS-класів і
постійно перемикатися між HTML та CSS блоками. Код став компактнішим — блок <style scoped> взагалі не потрібен,
всі стилі описуються прямо в шаблоні. Також Tailwind забезпечує єдиний стиль дизайну в усьому проєкті
завдяки готовим utility-класам. Проте є і недоліки, бо HTML-код стає перевантаженим великою кількістю класів,
що може ускладнювати читання, особливо для складних компонентів.
